# Воркфлоу после изменений кода

Выполнять автоматически после каждой задачи, без дополнительных просьб.

## Шаг 1 — Тесты (обязательно, без исключений)
- `grep` по имени функции/роутера → найти существующие тесты
- Новый endpoint/функция/компонент → минимум один тест
- Изменённое поведение → обновить существующий тест
- Тесты пишутся ДО запуска `test.sh`

## Шаг 2 — Запуск
```bash
bash scripts/test.sh          # только ошибки
bash scripts/test.sh --e2e    # E2E (требует docker compose)
```

## Шаг 3 — Фикс
Упали → найти причину → исправить → повторить Шаг 2. Максимум 3 итерации, затем сообщить пользователю.

## Шаг 4 — Ревью
Агент по `git diff HEAD`: логика, безопасность, edge cases, соответствие архитектуре. Критикал → фикс → Шаг 2.

## Шаг 5 — Коммит
Вывести в блоке кода: `<тип>(<scope>): <описание>`. `git commit` не делать автоматически.

---

## Замечания в процессе работы
Баг, уязвимость, техдолг, риск, архитектурное несоответствие → **сразу** записать в `.claude/NOTES.md`:
```
## [YYYY-MM-DD] <заголовок>
**Файл:** <путь>  **Тип:** <баг|техдолг|риск|улучшение|вопрос>
<описание>
---
```
